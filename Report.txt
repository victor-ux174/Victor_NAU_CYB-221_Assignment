### Why These Ports Matter in This Context

Open / listening ports remain one of the most important **initial reconnaissance indicators**:

- They allow attackers to fingerprint the operating system & software stack  
- Match versions against exploit databases (Exploit-DB, CVE details)  
- Target weak/default credentials or known vulnerabilities  
- Launch automated exploitation frameworks

In this case:

- Both ports belong to the **same application family** (VS Code)  
- Both are **intentionally loopback-bound**  
- Both exist solely to support **active local development workflow**  
- No unnecessary system services, web servers, databases, file shares, or remote access endpoints are listening

This is a textbook example of **least functionality** and **defense in depth** at the container level — Crostini + VS Code defaults are doing a very good job of minimizing unnecessary exposure.

### Security Controls and Hardening Recommendations

Even though risk is already very low, the following steps can reduce visibility further:

1. Close VS Code when not actively developing 
   → Immediately removes both TCP 5036 and 45017 (and any other ephemeral VS Code ports)

2. Limit VS Code features that open listeners
   - Disable Live Server, Live Preview, or any extension that starts an HTTP server  
   - Avoid running multiple VS Code windows simultaneously when possible  
   - Use `--disable-extensions` flag for particularly sensitive sessions

3. Check and clean ChromeOS Linux port forwarding rules
   Settings → Linux (Beta) → Port Forwarding →  
   → Delete any rules that are no longer needed (prevents accidental external exposure even if a port is later bound to 0.0.0.0)

4. Periodic monitoring (good hygiene)
   Run one of these commands regularly:  
   ```bash
   sudo ss -ltnup
   sudo ./port_inspector
   sudo lsof -iTCP -iUDP -n -P | grep LISTEN
   ```

5. Network-layer awareness
   - Use ChromeOS built-in VPN when on public Wi-Fi  
   - Be aware that Crostini networking is NAT’d through the host — most external exposure would require explicit port forwarding rules (which are off by default)
